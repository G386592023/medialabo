「Webコンテンツ作成」の成果

本テーマについて

このテーマでは、HTML、CSS、JavaScriptの基本を学び、これらを組み合わせてWebページを構成する方法を学ぶ。以下が主な学習内容である。

  • HTMLによってWebページの構造を記述する基本的な方法がわかる
  • CSSによってWebページの見た目を設定する基本的な方法がわかる
  • JavaScriptによってWebページのインタラクティブな処理を記述する基本的な方法がわかる
  • HTML、CSS、JavaScriptの構文リファレンスから、構文の書式や使用方法の情報を入手できる
  • HTML、CSS、JavaScript、画像・音声・動画ファイルを組み合わせて簡単な Web ページが作成できる

各授業回の学習事項

WWWと
HTML

　HTMLはWebページの構造を記述するための書式である。HTML文書を作ることによって、 Webページに掲載するコンテンツと、ページの基本的なレイアウトを設定できる。HTMLの要素はかなりの量がある為、 基本的に必要なものを都度調べながら使う。

演習の成果:
練習1-4: HTMLのさまざまな要素

CSSによるWeb
ページの装飾

　CSSとは、ページを装飾する記述である。一般に、CSSは要素の見た目を変更する為のものなので、HTMLを最初に与え、 次にCSSを記述する、という順番で書かれる。HTML同様CSSのプロパディは大量にあるため、都度調べながら使う必要がある。

演習の成果:
練習2-4: ウェブブラウザ説明ページの装飾

JavaScript
の基本

　JavaScriptはWebページ上で複雑な機能を実現するプログラミング言語である。JavaScriptプログラムは、Webページを 開いた直後やボタンを押したときに実行され、ページの自動更新やアニメーションなどの動きのあるページを作成する事ができる。

演習の成果:
練習3-6: オブジェクトの参照

DOMと
JavaScript

　DOMはWebコンテンツを表現するデータ構造である。DOMとJavaScriptを組み合わせて使用すると、Webコンテンツの内容を読み取ったり、 コンテンツに変更を加えたりする処理を書ける。また、今回の内容を利用し、動きのあるWebページを作成した。

演習の成果:
練習4-2~5: DOMプログラミングの演習

イベント処理
　

　イベントとは、Webページ上で起こった動作や出来事のことである。「ユーザーがキーを押す」、「エラーの発生」などが イベントにあたる。今回の内容を利用し、イベントが発生したときにJavaScriptプログラムが動作するような処理を実現した。

演習の成果:
練習5-3: イベントを選択する

感想

　はじめに、工夫点についてです。主に工夫したのはプロジェクト課題ですが、index.htmlも工夫しながら取り組みました。第2回で学習したフレックスボックスの設定を利用し、項目ごとにまとめた事が一番の工夫点であると思います。プロジェクト課題での工夫点は2つあります。一つ目は検索結果が存在しないチャンネル・ジャンルを選んでボタンを押した際に、「条件に一致する番組がありませんでした。」と表示するようにした点です。二つ目は検索する度に画像の背景色が全てランダムに変わるようなプログラミングを書いた点です。授業が始まった頃は番組毎にサムネイル画像を用意しようと思っていたのですが、いざURLを確認すると思った以上に番組が多いことがわかり、なるべく異なるサムネイルに見えるように前述の方法を取りました。また、評価には直接関わらないのですが、全体を通し見た目を整える事を意識しました。見た目をイメージ通りに設定するには様々な方法を調べ、設定の仕方を考える必要がある為、結果として工夫した箇所も増えたように思います。
　次に、苦心点です。主な苦心点は2つです。一つ目は課題4-2のオブジェクトをページ上に表示する課題です。JavaScriptにHTMLの要素を書き直している際に、いくつかのdiv要素を必要ないかと思い削除してしまい、CSSがうまく適用されずに大変なことになりました。githubで過去のプログラムを確認し、問題に気付けたのですが、以降プログラミングを消す際はより一層気をつけるようになりました。貴重な経験だったと思います。二つ目はプロジェクト課題の2回検索をしたときに1回目の検索結果を削除するプログラミングを書くことです。ここは一番苦戦した箇所でもあります。最後の課題だった事もあり、なるべく沢山プログラミングを書かずに該当処理を実行できないかと思った結果、逆に遠回りになってしまいました。最終的にはうまく動いたのですが、色々な方法を試し三日ほど悩みました。
　最後に感想です。総合的に、楽しく課題に取り組むことができたと思います。JavaScriptはJavaとは全く異なる言語だと聞いたことがあったので、この授業が始まるまで上手く扱えるか不安だったのですが、始まってみると条件式の書き方など似ている点も多くすんなりと理解できました。しかし、それでも見たことのない構文が出てくると、正常に動作するようプログラミングを書く作業に苦労しました。Webページの作成という題材は、動作を含めたレイアウトを考えデザインするという面からもかなり楽しかったので、今後も活用できるよう定期的に復習しておきたいです。
